# タスク完了時の手順

## 1. テスト実行（必須）
```bash
npm run test:ci          # ユニットテスト
npm run test:e2e         # E2Eテスト
```

すべてのテストが通過することを確認。

## 2. ビルド確認
```bash
npm run build
```

ビルドエラーがないことを確認。

## 3. Git操作
```bash
git add .
git commit -m "機能の概要

- 変更詳細1
- 変更詳細2
- テスト: XX個追加（合計YY個）"
```

コミットメッセージは日本語で、変更内容を明確に記載。

## 4. デプロイ（任意）
`master`ブランチへpushすると、GitHub Actionsが自動的に：
1. テスト実行
2. ビルド
3. GitHub Pagesへデプロイ

テスト失敗時はデプロイ中止。

## 重要な注意事項
- テスト実装前にコード変更しない（リグレッション防止）
- 既存のテストパターンを確認してから新規テストを作成
- CLAUDE.md、README.mdを必要に応じて更新
